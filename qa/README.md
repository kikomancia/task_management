# ✅ QA — VALIDATION RULES

## Functional Requirements

### Task Creation
- ✅ Task title is required (non-empty)
- ✅ Task description is optional
- ✅ New tasks default to "todo" status
- ✅ Task timestamps (createdAt, updatedAt) set automatically
- ✅ Task ID generated by MongoDB
- ✅ Form clears after successful submission

### Task Retrieval
- ✅ All tasks loaded on app startup
- ✅ Tasks displayed in correct columns based on status
- ✅ Tasks persist after browser refresh
- ✅ Empty columns display "No tasks" message

### Task Updates
- ✅ Task status updates when moved between columns
- ✅ updatedAt timestamp changes on status update
- ✅ All tasks remain visible after update

### Task Deletion
- ✅ Task removed from column immediately
- ✅ Task removed from database
- ✅ Cannot undo deletion (MVP - no undo feature)

### Layout
- ✅ 3 columns visible simultaneously (desktop)
- ✅ All columns equal width
- ✅ Column headers display correctly
- ✅ Task cards render in correct columns

### Drag & Drop
- ✅ Tasks can be dragged within columns
- ✅ Tasks can be dragged between columns
- ✅ Drop target highlights on drag over
- ✅ Task status updates after drop
- ✅ No visual artifacts after drag/drop

- ✅ Consistent spacing and padding
- ✅ Buttons have hover effects

## Performance Validation

- ✅ App loads in < 3 seconds
- ✅ Drag & drop is smooth (60 FPS)
- ✅ API responses < 500ms
- ✅ No memory leaks in React components
- ✅ No console errors

## Data Validation

### Input Validation
- ✅ Title max length: 255 characters
- ✅ Description max length: 2000 characters
- ✅ Status only accepts: "todo", "in-progress", "done"
- ✅ Invalid inputs rejected with error message

### Database Validation
- ✅ All required fields present in document
- ✅ Status values match enum
- ✅ Timestamps are valid ISO 8601 format
- ✅ ObjectId format valid

## API Validation

### GET /api/tasks
- ✅ Returns 200 status
- ✅ Returns array of tasks
- ✅ Empty array if no tasks
- ✅ Returns 500 if database error

### POST /api/tasks
- ✅ Returns 201 status on success
- ✅ Returns created task with _id
- ✅ Returns 400 if title missing
- ✅ Returns 400 if invalid status
- ✅ Returns 500 if database error

### PUT /api/tasks/:id
- ✅ Returns 200 status on success
- ✅ Returns updated task
- ✅ Returns 400 if invalid status
- ✅ Returns 404 if task not found
- ✅ Returns 500 if database error

### DELETE /api/tasks/:id
- ✅ Returns 200 status on success
- ✅ Returns deleted task object
- ✅ Returns 404 if task not found
- ✅ Returns 500 if database error
